What are microcontrollers and why should you care? #

Microcontrollers are very small computers that are programmed to be good at doing one job and one job only. Mostly they come in the form of an IC (Integrated Circuits) and have several pins protruding from them, they even may have a whole pcb dedicated to them. Over the years the development of such microcontrollers have been so successful that now we can even have bluetooth, wifi and several other capabilities built right onto these little pcbs. Using such boards and programming your own software you can make several new projects and machines that can help you and others.

What’s an arduino? #

One of the most popular series of microcontrollers is the Arduino series of open-source microcontrollers. Later in this article we will build an Arduino Uno from scratch and program it with the help of the C language (Arduino uses a special flavour of C, we can call it Arduino C).

The Blink Sketch: Hello World of Arduino #

One of the most beginner-friendly sketches (a sketch is a program written in Arduino C for some Arduino board/microcontroller) is the Blink sketch that comes with the Arduino IDE. If you don’t have the IDE installed go ahead and follow instructions for your OS from this link.

Get your hands dirty! #

For this tutorial you will be needing an Arduino microcontroller board, we suggest the Arduino Uno board, as it is both beginner friendly as well as has the perfect degree of capability for learning!

Now launch the Arduino IDE and go to File -> Examples -> 01Basics and then finally the Blink sketch. Plug in your USB cable to your Arduino Uno board and then plug it into the computer. Now we have to check whether we have the right board and com-port selected for the arduino board, for that go to Tools and make sure you have selected Arduino Uno under the boards and have selected the correct Serial Port where the Arduino is connected. Now simply press the upload button and wait until the status LED has stopped blinking on your board (the status LED is an LED on your arduino uno board that blinks and lets the user know that a program is being uploaded to the board, and it stops blinking once the code is uploaded). You can finally see the result of all of the actions above, i.e if done properly an LED connected to your arduino board should be blinking for precisely a second now.

Understanding the code #

Now that you have successfully coded your first ever arduino project, let us understand the code behind the blinking LED. If you are familiar with the C programming language you might know that in the C language, in any program we have a mandatory main function, in Arduino C however, there is not one, but two mandatory parts required: a) the void setup function and b) the void loop function. All arduino sketches must include these two lines of code otherwise they cannot be compiled.

Now you must be asking yourself, how does this work? Everytime a program is running the variables created at the top of the code are initialized, then the void setup code is run once, all of the functions inside this part is executed once, then the loop function is called, the code inside the loop function runs until you power off the Arduino board or the board is reset.

Inside the setup function the first thing we do is initialize all necessary variables, that is precisely what the code is doing through the pinMode(13,OUTPUT) line. pinMode is a function which helps us to set our desired pin (the several i/o pins coming from the board) as an output or input, in this case we set it as an output, (pin 13 is connected to the built-in LED, you can also connect an external LED, make sure connect the positive end to pin 13 on the board and the negative to any pin labelled GND, which stands for ground). In your sketch you may also find that the code is written as pinMode(LED_BUILTIN, OUTPUT) which is perfectly fine and just states the same thing (arduino ide is setting the variable LED_BUILTIN as an output and it knows that in an arduino uno that pin is pin 13). You can change the code and play around with different pins, at this point you may find out not all pins work alike!

Now in the loop function we have two other functions digitalWrite and delay. digitalWrite is a function that helps us write a state to the assigned pin. In the first line i.e. pinMode(13, HIGH) we set the pin 13 to HIGH, which is 5v. Next we pause the program for a specific time using the delay function. The parameter is in milliseconds, so 1000 millisecond is equal to a delay of 1 second. During this delay the program does nothing and the LED stays on for this time. Next, we again call the pinMode function and turn the LED off, by setting pin 13 to LOW or 0v. Again we set a delay and for that time period the LED stays off.

If the delay function is confusing to you try and run the code deleting the two delays, you may notice that the LED stays on, but is somehow a lot dimmer. This is happening because the LED is turning on and off very fast, so fast that you cannot even notice it. Without delays the processor is executing the lines very fast, and we don’t want that, hence we must include the delays.

One of the most important things to remember is to never use hard-code. What I mean by this is, say you want to use pin 4 as an output, and you have already written the program using pin 13. Now you will have to go in and change all occurrences of the pin 13. In order to curb this we simply initialize a new variable and set it to the proper pin number we want, that way we can change that pin by only changing a single line of code! See the below example:

 1int ledPin = 4;		//initialise a variable for the pin connected to the LED
 2void setup(){
 3	pinMode(ledPin, OUTPUT);	//set pin as OUTPUT
 4}	
 5void loop(){
 6	digitalWrite(ledPin, HIGH);		//turn LED on(5v)
 7	delay(1000);					//delay of 1 sec
 8	digitalWrite(ledPin,LOW);		//turn LED off(5v)
 9	delay(1000);					//delay of 1 sec
10}

You can change the pin and delay values and play around with it and try different things! Even plug different LEDs and turn them on or off in a specific pattern. Now that you have the basics covered let’s try to make our own arduino uno board!

Let’s make an Arduino Uno from scratch! #

With very little components we can easily make our own arduino uno! The Arduino Uno uses the Atmega328p microcontroller as the brains of it and we will be seeing how to assemble and upload code to it! Let’s look at the parts we will be needing now:

  • breadboard
  • jumper wires
  • atmega328p microcontroller
  • 16mHz crystal
  • 2x 22\(pF\) capacitors
  • 10\(\mu F\) capacitor
  • 100\(pF\) capacitor
  • 10k resistor
  • LED of any preferred colour
  • Arduino Uno and USB cable
  • FTDI Programmer

Barebones Arduino Uno on a Breadboard #

First and foremost you must know how a breadboard works! In most of the breadboards you can see two parallel rails at either end of the board and several rails in the inside of the rectangle. The two outer rails are power and ground rails. The inner rails are connected vertically, see the below image:

The gap in the middle is used to place ICs so that the pins do not connect to each other. So first place the Atmega IC in the middle of the board, now you have to know the pin-out of this particular IC, for that refer to the below diagram:

For the sake of simplicity we will only be going through the necessary pins for this tutorial, they are:

  1. VCC / Pin 7 and 20
  2. GND / Pin 8 and 22
  3. Reset / Pin 1
  4. PB6 and PB7 / Pins 9 and 10 (crystal)
  5. Tx / Pin 3
  6. Rx / Pin 2

First connect a 10k resistor to the Reset pin (Pin 1) of the Atmega and 5v or the power rail, this prevents the microcontroller to reset itself. Now connect the 16mHz crystal to pins 9 and 10, and connect 22 \(pF\) capacitor to each pin and GND. Now connect Pins 7 and 20 to 5v and Pins 8 and 22 to GND. Also connect a 10 \(\mu F\) capacitor to VCC and GND. Now we will be connecting an LED to Pin 19 and GND, this will be the status LED. Refer to the below diagram:

This is the barebones arduino uno circuit! Now to upload code to it you can go several directions, but one of the easiest way is to use an FTDI programmer. Connect Tx to Rx and Rx to Tx and DTR to Pin 1 through another 100 \(pF\) capacitor and VCC to VCC and GND to GND.

Now open up the blink sketch and upload the sketch! If everything is done properly the LED should be blinking in the breadboard just like you did in the arduino board!

Burning the Bootloader onto a Barebones Arduino #

If however, things don’t work out, you might have a missing bootloader. A bootloader is a special software that always resides on the chip to make sure code can be uploaded to it. Most chips come with a bootloader pre-loaded, if something messes it up however, we can also burn the bootloader onto it! To burn a bootloader onto your breadboard arduino first make the preliminary circuit that we mentioned earlier (mentioned just before the FTDI connections). You don’t need to connect anything to the FTDI now, instead we will be needing the arduino Uno board for this. First, open up the Arduino IDE and click on File -> Examples -> 11.ArduinoISP -> ArduinoISP. This sketch will help us burn the bootloader onto the Atmega328p chip on the breadboard. Click on the upload button and let it finish.

Now connect Pin 13 of Arduino Uno board to Pin 19 of Atmega, Pin 12 of Arduino to Pin 18 of Atmega, Pin 11 to Pin 17 and Pin 10 to Reset Pin / Pin 1 of Atmega through a 100\(pF\) capacitor.

Finally connect 5v of Arduino to VCC of Atmega and GND to GND. Now go to Tools -> Programmer -> Arduino as ISP. Then Tools -> Burn Bootloader. If successful you will see a message on the console. Another way to confirm that the chip has the bootloader installed is that an LED connected to Pin 19 on the Atmega will blink everytime a sketch is being uploaded to it.
