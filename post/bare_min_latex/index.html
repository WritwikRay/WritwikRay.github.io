In this article I encourage you to start off learning LaTex by covering only the very basics, the bare-minimum. Starting off with the bare-minimum is one of the best ways to learn something new. (For me, personally it is always the best possible way, there’s no way I can procrastinate, I will feel motivated once I finish that day’s lesson, since most of the time the bare-minimum is not much to learn in a day!)

Let’s begin! #

Now the very first thing you need to know is that LaTex needs to be installed in your computer, to do that head over to the official website and follow directions for your own OS. The second thing is the editor of choice. You can use any text-editor of your preference, however it is nice to know that there are several “tex-editors” also available (see what I did there!) One of the best editors is Kile! And if you’re on Windows…then maybe try Linux for a spin?!?! Kidding, you can use TexStudio. Whatever is your choice, learn the basic controls of that editor first before delving into learning LaTex. By this what I mean is this: No matter what editor you use, you have to know that LaTex is a compiled language, meaning, you first write some code, then you run a compiler over it (by hitting the compile button on the editor controls), and you have a pdf file generated (or maybe just a preview generated).

I would recommend beginners to start out with either Kile or Overleaf, as both of them are very beginner friendly and easy to use.

Creating your first document #

Now let’s get started (for real). Every LaTex document starts out with the following code:

\documentclass{article}
\begin{document}
This is a test.
\end{document}

Go ahead and compile the document! Great, you have now created your first-ever LaTex document. Now let’s actually unbderstand what this bit of code is doing. Every LaTex file is primarily divided into two portions: 1. the preamble and 2.the document portion or the actual content of the document. The preamble is everything inbetween the \documentclass declaration and the \begin{document} portion. In the example above we can’t really see the preamble portion, but there we will declare all the libraries we will be using in the document (more on this on some later post!). What are libraries you ask? well libraries are pre-written bits of code that will help us write less code to achieve some extra functionalities in our LaTex documents. Depending on your machine and LaTex installation you may need to install libraries to run some extra functionalities. We will come to this soon.

The second portion of any LaTex document is the actual content of the document, this resides between the \begin{document} and the \end{document} tags. In the example above we have simply added one line of text to show in the final pdf.

Let’s first concentrate on the arrangement of lines. You will notice that whitespace is ignored in LaTex. What I mean byu this is the following bit of code will generate the same output as the one above.

1	\documentclass{article}
2	\begin{document}
3						This is a test.
4	\end{document}
